O que é um Data Warehouse?

Definição e Propósito

Um Data Warehouse (DW) é um repositório centralizado de dados estruturados, projetado especificamente para consulta e análise — não para operações do dia a dia. Ele integra dados de múltiplas fontes, aplica transformações e os organiza para apoiar a tomada de decisão.

💡 William Inmon (1992) — "Pai do DW" Definiu DW como uma coleção de dados orientada por assunto, integrada, variante no tempo e não volátil, criada para apoiar processos de tomada de decisão gerencial.
✅ Ralph Kimball — Abordagem Dimensional Propôs a construção bottom-up com Data Marts independentes interligados por bus de dimensões conformed. É a base do modelo estrela que estudamos na Semana 03.
As 4 Características Fundamentais (Inmon)

Pilares do Data Warehouse

Clique em cada característica para expandir:

🎯
Orientado por Assunto
Os dados são organizados em torno de temas de negócio (Vendas, Clientes, Produtos) — não em torno de aplicações (ERP, CRM). Facilita análises focadas em áreas específicas do negócio.
🔗
Integrado
Dados de múltiplas fontes heterogêneas são unificados com convenções padronizadas: unidades de medida, nomenclaturas, codificações. Ex: "M/F" em um sistema se torna "Masculino/Feminino" no DW.
⏰
Variante no Tempo
O DW mantém histórico de dados ao longo do tempo (geralmente 5–10 anos). Cada registro tem um timestamp. Diferentemente dos sistemas OLTP que mantêm apenas o estado atual.
🔒
Não Volátil
Dados carregados no DW não são atualizados ou deletados individualmente — apenas adicionados. Isso garante integridade histórica. As únicas operações são: carga inicial e acesso para leitura.
Evolução Histórica

Como chegamos até aqui?

1960
Decision Support Systems (DSS)
Primeiras tentativas de separar dados analíticos dos sistemas transacionais.
1988
IBM Information Warehouse
Barry Devlin e Paul Murphy publicam o conceito de "business data warehouse" na IBM Systems Journal.
1992
Inmon publica "Building the Data Warehouse"
Livro seminal que define formalmente os conceitos de DW e lança a arquitetura top-down (CIF).
1996
Kimball lança "The Data Warehouse Toolkit"
A abordagem dimensional (estrela/snowflake) e o conceito de Data Mart tornam-se mainstream.
2010+
Cloud DW: Redshift, BigQuery, Snowflake
O DW migra para a nuvem, com escalabilidade elástica e separação entre armazenamento e computação.

Data Mart
Subconjunto do DW focado em uma área de negócio específica (ex: Marketing DM, Finanças DM). É mais rápido de implementar e serve como porta de entrada para o DW completo.

ETL

+5 XP

OLAP
Online Analytical Processing — tecnologia para consultas multidimensionais complexas com operações como drill-down, roll-up, slice e dice sobre cubos de dados.
+5 XP
⚡
OLTP
Online Transaction Processing — sistemas operacionais que processam transações do dia a dia (vendas, pedidos, cadastros). Otimizados para inserção/atualização frequente.
+5 XP
🗄️
Staging Area
Área temporária onde os dados extraídos são armazenados antes da transformação. Funciona como buffer entre as fontes e o DW final, isolando o ambiente de produção.
+5 XP
🏗️
Metadata
Dados sobre os dados — descrevem a estrutura, origem, significado e regras de negócio dos dados no DW. O repositório de metadata é essencial para governança e rastreabilidade.

🔀 OLTP vs Data Warehouse

Arraste cada característica para a coluna correta. OLTP são sistemas transacionais (ERP, e-commerce). Data Warehouse é analítico.

📦 Características disponíveis:

Dados em tempo real
Histórico de 5–10 anos
Muitas gravações por segundo
Consultas analíticas complexas
Schema altamente normalizado (3NF)
Schema desnormalizado (Star/Snowflake)
Foco em poucos registros por transação
Foco em agregações e tendências
Dados de múltiplas fontes integrados
Alta concorrência de usuários simultâneos
⚡ OLTP — Sistemas Transacionais
🏢 Data Warehouse — Analítico

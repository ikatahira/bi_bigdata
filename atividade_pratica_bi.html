Módulo A

Limpeza e transformação de dados

Os alunos recebem uma planilha com dados "sujos" e aplicam as etapas de um pipeline real: identificar inconsistências, padronizar e preparar para análise.

30' duração

Objetivos de aprendizagem

Identificar dados ausentes, duplicados e fora de padrão
Usar filtros e formatação condicional para visualizar problemas
Padronizar texto com MAIÚSCULA, ARRUMAR, SUBSTITUIR
Remover duplicatas e documentar o processo de limpeza

Roteiro passo a passo

1
Abrir a planilha modelo

Faça o download do arquivo .csv abaixo. No Google Sheets: Arquivo → Importar → selecione o arquivo → Substituir planilha. Em seguida, Arquivo → Fazer uma cópia para trabalhar na sua versão.

2
Inspecionar os dados

Role toda a planilha sem editar nada. Na aba "Observações" (crie uma nova aba com esse nome), anote pelo menos 3 problemas que você encontrou: campos vazios, erros de digitação, formatos diferentes, valores inválidos.

3
Aplicar formatação condicional

Selecione a coluna C (Cidade). Vá em Formatar → Formatação condicional → Adicionar nova regra → "A célula está vazia" → escolha a cor vermelha como destaque. Repita para a coluna F (Data).

4
Padronizar os nomes com fórmula

Na coluna H (auxiliar, crie um cabeçalho "Nome_limpo"), use: =MAIÚSCULA(ARRUMAR(B2)) para padronizar nomes. Arraste para todas as linhas. Depois selecione a coluna H → copiar → colar especial como Valores → substitua a coluna B.

5
Corrigir abreviações de cidades

Use Editar → Localizar e substituir (Ctrl+H). Substitua "RJ" por "Rio de Janeiro" e "POA" por "Porto Alegre". Marque a opção "Diferenciar maiúsculas" para evitar substituições incorretas.

6
Remover duplicatas

Selecione toda a tabela (Ctrl+A). Vá em Dados → Limpeza de dados → Remover cópias. Uma janela mostrará quantas linhas duplicadas existem. Anote esse número na aba Observações.

7
Documentar o resultado

Na aba "Observações", complete: (a) quantos registros havia originalmente; (b) quantos problemas foram encontrados e de que tipo; (c) quantos registros restaram após a limpeza; (d) qual é a qualidade final dos dados (0–10).

Módulo B

Fórmulas e análise de dados

Os alunos usam fórmulas essenciais do Google Sheets para extrair métricas de uma base de vendas: totais, médias, contagens condicionais e rankings.

35' duração

Objetivos de aprendizagem

Calcular totais e médias com SOMA e MÉDIA
Usar SOMASE e CONT.SE para análises condicionais
Criar rankings com MAIOR e identificar top performers
Categorizar desempenho usando a função SE

Roteiro passo a passo

1
Abrir a planilha de vendas

Faça o download e importe no Google Sheets. A planilha contém 15+ registros de vendedores com: ID, Vendedor, Região, Produto, Valor de Vendas (R$) e Mês.

2
Criar a aba de análise

Adicione uma nova aba chamada "Análise". Monte uma tabela com os seguintes rótulos: Total de Vendas, Média por Vendedor, Maior Venda, Menor Venda, Nº de Transações.

3
Preencher as métricas gerais

Na aba Análise, use: =SOMA(Dados!E:E) para total, =MÉDIA(Dados!E:E) para média, =MÁXIMO(Dados!E:E) para maior venda e =CONT.VALORES(Dados!A:A)-1 para contar transações.

4
Vendas por região com SOMASE

Crie uma tabela com as 3 regiões. Para cada uma use: =SOMASE(Dados!C:C,"Sudeste",Dados!E:E) — substituindo o nome da região. Verifique se a soma das regiões é igual ao total geral.

5
Classificar vendedores por desempenho

Nos dados originais, crie uma coluna F chamada "Nível". Use: =SE(E2>=2000,"Alto","Baixo"). Depois use =CONT.SE(F:F,"Alto") para contar quantos são de alto desempenho.

6
Ranking — top 5 maiores vendas

Na aba Análise, crie uma tabela "Top 5". Use: =MAIOR(Dados!E:E,1) para o 1º lugar, =MAIOR(Dados!E:E,2) para o 2º, e assim por diante até o 5º.

7
Validar e interpretar

Responda na aba Análise: (a) Qual região tem maior volume de vendas? (b) Qual porcentagem de vendedores é de alto desempenho? (c) Existe diferença entre os meses de Março e Fevereiro?

Módulo C

Tabela dinâmica (Pivot Table)

Os alunos usam tabelas dinâmicas para resumir grandes volumes de dados, cruzar dimensões e identificar padrões — habilidade central em BI.

40' duração

Objetivos de aprendizagem

Criar uma tabela dinâmica a partir de dados brutos
Configurar linhas, colunas, valores e filtros corretamente
Comparar vendas cruzando Região × Produto
Interpretar os resultados e tirar conclusões analíticas

Roteiro passo a passo

1
Preparar os dados

Importe o arquivo do Módulo C. Certifique-se de que todos os cabeçalhos estão preenchidos na linha 1 e não há linhas em branco entre os dados. A tabela dinâmica não funciona bem com dados desorganizados.

2
Inserir a primeira tabela dinâmica

Clique em qualquer célula dentro dos dados. Vá em Inserir → Tabela dinâmica → Nova planilha. Isso cria automaticamente a aba "Tabela dinâmica 1" com o painel lateral de configuração.

3
Configurar Região × Produto

No painel lateral: arraste "Região" para Linhas. Arraste "Produto" para Colunas. Em Valores, clique em Adicionar → selecione "Vendas (R$)" → a função deve ser SOMA. Você verá uma tabela cruzada se formando.

4
Adicionar filtro de mês

No painel, arraste "Mês" para Filtros (parte superior). Um filtro aparece acima da tabela. Selecione apenas "Março" e observe como os totais mudam. Anote os valores. Depois selecione "Fevereiro" e anote novamente.

5
Segunda tabela dinâmica — por contagem

Crie uma nova tabela dinâmica (Inserir → Tabela dinâmica → Nova planilha). Desta vez: "Vendedor" nas Linhas, "Vendas (R$)" nos Valores mas com função CONT (não SOMA). Isso mostra quem realizou mais transações.

6
Calcular ticket médio

Numa terceira coluna da segunda tabela dinâmica, use uma fórmula manual dividindo SOMA pelo CONT para cada vendedor. Isso revela o ticket médio por vendedor — diferente do volume total.

7
Análise e conclusões

Crie uma aba "Conclusões" e responda: (a) Qual região vendeu mais em valor? (b) Qual produto tem melhor desempenho no Sul? (c) Qual vendedor tem maior ticket médio (não necessariamente o maior volume)? (d) Por que ticket médio pode ser mais relevante que volume total?

Módulo D

Dashboard e visualização

Os alunos criam um dashboard executivo com gráficos interligados, segmentações e KPIs — unindo tudo que foi aprendido nos módulos anteriores.

45' duração

Objetivos de aprendizagem

Criar gráficos de linhas, barras e pizza a partir de tabelas dinâmicas
Usar Segmentação de dados para filtros interativos
Organizar um painel visual com KPIs, gráficos e filtros
Escolher o tipo de gráfico correto para cada tipo de dado

Roteiro passo a passo

1
Preparar a aba Dashboard

Crie uma nova aba chamada "Dashboard". Para deixar o visual limpo: Exibir → Linhas de grade → desmarque "Mostrar linhas de grade". Use Formatar → Planilha → Alterar cor para identificar visualmente a aba.

2
KPIs no topo — 4 cartões

Nas células B2:E4, crie 4 blocos com fórmulas apontando para a aba Análise: Total de Vendas, Ticket Médio, Nº de Transações, Melhor Região (use texto manual). Formate com fundo colorido e fonte maior para destacar.

3
Gráfico de colunas — vendas por região

Selecione a mini-tabela de totais por região (da aba Análise) → Inserir → Gráfico. O Sheets sugere automaticamente. Mude para "Gráfico de colunas" se necessário. Clique no gráfico e arraste para o Dashboard.

4
Gráfico de linhas — evolução mensal

Crie uma tabela auxiliar com Mês e Total de Vendas (usando SOMASE por mês). Selecione essa tabela → Inserir → Gráfico de linhas. Este gráfico mostra tendência ao longo do tempo. Mova para o Dashboard.

5
Gráfico de pizza — mix de produtos

Crie uma tabela com Produto e Total de Vendas. Insira um gráfico de pizza. Atenção: pizzas funcionam bem com poucos fatias (3–5). Com muitos produtos, prefira barras horizontais.

6
Adicionar Segmentação de dados

Clique na tabela dinâmica (aba da tabela dinâmica) → no menu Dados → Adicionar segmentação → selecione "Região". Copie a segmentação e cole no Dashboard. Ao clicar em uma região, todos os gráficos vinculados filtram automaticamente.

7
Reflexão final e apresentação

Responda na aba "Conclusões": (a) Por que você usou barras para regiões e não pizza? (b) O que mudaria no dashboard se o público fosse a equipe operacional (não executivos)? (c) Que dado importante está faltando neste dashboard?

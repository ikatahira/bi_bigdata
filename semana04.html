📚 Conteúdo Teórico

OLAP — Online Analytical Processing

OLAP é uma tecnologia para responder consultas analíticas multidimensionais com velocidade. Diferente do OLTP (que registra transações), o OLAP serve para analisar grandes volumes de dados históricos.

🏦
OLTP vs OLAP
Clique para expandir
OLTP: muitas transações pequenas, dados do momento, altamente normalizado.

OLAP: poucas consultas pesadas, dados históricos, desnormalizado para velocidade. São complementares.
⚡
Por que OLAP?
Clique para expandir
Uma consulta como "Vendas do Q3/2023 na região Sul por categoria?" pode levar horas em OLTP. Em OLAP, a resposta vem em milissegundos porque os dados já foram pré-agregados.
🗄️
Pré-agregação
Clique para expandir
O cubo OLAP pré-calcula somas, médias e contagens para todas as combinações de dimensões, eliminando o custo computacional no momento da consulta.

Um cubo OLAP é uma estrutura multidimensional. Cada eixo é uma dimensão, e cada célula guarda um valor agregado (medida).

// Cubo de Vendas — estrutura conceitual
Dimensões:
  ├── Tempo     → Ano > Trimestre > Mês > Dia
  ├── Produto   → Categoria > SubCategoria > Produto
  └── Região    → País > Estado > Cidade

Medidas (células do cubo):
  ▸ Receita Total     → R$ 142.000
  ▸ Qtd Vendida       → 1.847 unidades
  ▸ Ticket Médio      → R$ 76,88

// Célula = interseção de valores de cada dimensão
// Ex.: Receita de Notebook em SP em Jan/2024 = R$ 52.000

Explore cada operação clicando nos botões:

🧊
MOLAP
Multidimensional OLAP
Dados em estrutura proprietária multidimensional. Muito rápido para consultas, mas limitado em volume. Ex.: Microsoft SSAS.
🗃️
ROLAP
Relational OLAP
Usa banco relacional. Suporta grandes volumes. Consultas geradas em SQL dinamicamente. Ex.: Tableau + PostgreSQL.
⚖️
HOLAP
Hybrid OLAP
Combina os dois: dados detalhados no BD relacional, agregações no cubo multidimensional. Melhor custo-benefício.

Questão 1 de 5
Explique com suas palavras a diferença entre OLTP e OLAP. Por que um banco de dados transacional não é adequado para análises de BI?
✅ Gabarito — Resposta esperada
OLTP (Online Transaction Processing) é projetado para operações do dia-a-dia: muitas transações pequenas e rápidas (INSERT/UPDATE/DELETE). Os dados são atuais e altamente normalizados.

OLAP (Online Analytical Processing) é projetado para análises: poucas consultas pesadas sobre grandes volumes de dados históricos. Os dados são desnormalizados para acelerar leituras.

Um BD transacional não é adequado para BI porque: (1) consultas analíticas com múltiplos JOINs em tabelas normalizadas são muito lentas; (2) ele guarda apenas o estado atual, sem histórico adequado; (3) consultas pesadas travam o sistema operacional do negócio.

Questão 3 de 5
Explique as operações Drill-down e Roll-up com um exemplo concreto. O que elas têm em comum? O que as diferencia?
✅ Gabarito
Ambas navegam pela hierarquia de uma dimensão — a diferença é a direção.

Roll-up ⬆ sobe na hierarquia, agregando: de Mês para Trimestre, de Cidade para Estado. Você resume os dados, perdendo detalhe mas ganhando visão geral.

Drill-down ⬇ desce na hierarquia, detalhando: de Trimestre para Mês, de Estado para Cidade. Você expande os dados para investigar algo específico.

Exemplo: Ao analisar vendas por Ano e perceber que 2023 foi fraco, você faz Drill-down para Trimestre para ver qual período foi ruim. Depois Drill-down novamente para Mês para identificar o mês exato.

Questão 4 de 5
Qual a diferença entre as operações Slice e Dice? Dê um exemplo de cada usando um cubo de dados climáticos (temperatura, umidade, vento).
✅ Gabarito
Slice ✂ — Fixa uma dimensão em um único valor, reduzindo o cubo em uma dimensão (3D vira 2D).
Exemplo climático: Slice(Mês = Janeiro) → temperatura/umidade/vento de Janeiro em todas as cidades e anos.

Dice 🎲 — Filtra múltiplas dimensões com intervalos ou listas. O resultado é um sub-cubo (ainda multidimensional).
Exemplo climático: Dice(Mês ∈ {Dez,Jan,Fev} AND Cidade ∈ {SP,RJ}) → sub-cubo climático do verão em SP e RJ.

Resumo: Slice é um corte único em uma dimensão. Dice é filtro em múltiplas dimensões mantendo a estrutura.

Questão 5 de 5
Compare MOLAP, ROLAP e HOLAP: em qual cenário você escolheria cada um? Justifique com base em volume de dados e necessidade de performance.
✅ Gabarito
MOLAP — Quando a performance é crítica e o volume é controlado (até alguns GB). Resposta em milissegundos. Ideal para dashboards executivos com KPIs críticos.

ROLAP — Quando o volume é muito grande (TB ou PB) e flexibilidade de queries SQL é importante. Aceita-se performance menor. Ex.: DW com 10 anos de histórico de uma rede varejista nacional.

HOLAP — Quando precisa do melhor dos dois: dados detalhados em grande volume (ROLAP) + agregações rápidas pré-calculadas (MOLAP). Mais complexo de manter. Ex.: sistema de BI de uma operadora de telecomunicações.

⚠️ Pull-up DHT11: conecte um resistor 10kΩ entre o pino DATA e o 5V.
⚠️ O LDR forma um divisor de tensão com o resistor 10kΩ — A0 lê a tensão no nó central.
💡 Quanto mais luz, menor a resistência do LDR, maior a leitura em A0 (0–1023).
💻 Código Completo — Semana 04
Arduino C++ — estacao_climatica_s04.ino 
/*
 * BDN007 - Semana 04: Estacao Climatica Multidimensional
 * Projeto Evolutivo - Parte 1: Coleta de Dados
 *
 * Conceito OLAP aplicado:
 *   MEDIDAS   -> temperatura, umidade, luminosidade
 *   DIM TEMPO -> millis() / timestamp serial
 *   DIM LOCAL -> fixo (sala de aula) - expandido na S05
 *
 * Biblioteca necessaria: DHT sensor library (Adafruit)
 * Instalar: Arduino IDE -> Ferramentas -> Gerenciar Bibliotecas
 */

#include <DHT.h>

// ── Configuracao dos pinos ────────────────────
#define DHTPIN     2      // Pino digital do DHT11
#define DHTTYPE    DHT11  // Modelo do sensor
#define LDRPIN     A0     // Pino analogico do LDR
#define INTERVALO  5000   // Coleta a cada 5 segundos

// ── Objetos e variaveis ───────────────────────
DHT dht(DHTPIN, DHTTYPE);

unsigned long ultimaLeitura = 0;
int contadorLeitura = 0;

// Estrutura para guardar uma leitura (celula do cubo)
struct LeituraCubo {
  unsigned long timestamp;    // DIM_TEMPO
  float         temperatura;  // MEDIDA 1
  float         umidade;      // MEDIDA 2
  int           luminosidade; // MEDIDA 3
  String        local;        // DIM_LOCAL (fixo por ora)
};

// Buffer circular das ultimas 10 leituras
LeituraCubo buffer[10];
int bufferIdx = 0;

// ── setup() ──────────────────────────────────
void setup() {
  Serial.begin(9600);
  dht.begin();

  Serial.println("================================");
  Serial.println("BDN007 - Estacao Climatica S04");
  Serial.println("Cubo: Temperatura x Umidade x Luz");
  Serial.println("================================");
  Serial.println("ID | Tempo(ms) | Temp(C) | Umid(%) | Luz(0-1023) | Local");
  Serial.println("---+----------+---------+---------+-------------+------");

  delay(2000); // DHT11 precisa de estabilizacao
}

// ── loop() ───────────────────────────────────
void loop() {
  unsigned long agora = millis();

  if (agora - ultimaLeitura >= INTERVALO) {
    ultimaLeitura = agora;

    // 1. Ler sensores
    float temp = dht.readTemperature();
    float umid = dht.readHumidity();
    int   luz  = analogRead(LDRPIN);

    // 2. Validar leituras (DHT11 pode falhar)
    if (isnan(temp) || isnan(umid)) {
      Serial.println("ERRO: Falha na leitura do DHT11!");
      return;
    }

    // 3. Criar celula do cubo
    LeituraCubo celula;
    celula.timestamp    = agora;
    celula.temperatura  = temp;
    celula.umidade      = umid;
    celula.luminosidade = luz;
    celula.local        = "Sala-A1"; // DIM_LOCAL fixo (S05 expande)

    // 4. Armazenar no buffer circular
    buffer[bufferIdx % 10] = celula;
    bufferIdx++;
    contadorLeitura++;

    // 5. Imprimir no Serial Monitor (formato CSV)
    Serial.print(contadorLeitura);
    Serial.print("  | ");
    Serial.print(agora);
    Serial.print("   | ");
    Serial.print(temp, 1);
    Serial.print("C  | ");
    Serial.print(umid, 1);
    Serial.print("%   | ");
    Serial.print(luz);
    Serial.print("         | ");
    Serial.println(celula.local);

    // 6. A cada 10 leituras: exibir mini-analise OLAP (Roll-up)
    if (contadorLeitura % 10 == 0) {
      exibirAnaliseOLAP();
    }
  }
}

// ── Analise OLAP local (Roll-up das 10 ultimas) ──
void exibirAnaliseOLAP() {
  float somaTemp = 0, somaUmid = 0;
  float maxTemp  = -999, minTemp = 999;
  int   somaLuz  = 0;

  for (int i = 0; i < 10; i++) {
    somaTemp += buffer[i].temperatura;
    somaUmid += buffer[i].umidade;
    somaLuz  += buffer[i].luminosidade;
    if (buffer[i].temperatura > maxTemp) maxTemp = buffer[i].temperatura;
    if (buffer[i].temperatura < minTemp) minTemp = buffer[i].temperatura;
  }

  Serial.println("\n─── ROLL-UP: Resumo das ultimas 10 leituras ───");
  Serial.print("  Temp Media : "); Serial.print(somaTemp / 10.0, 1); Serial.println("C");
  Serial.print("  Temp Max   : "); Serial.print(maxTemp, 1);          Serial.println("C");
  Serial.print("  Temp Min   : "); Serial.print(minTemp, 1);          Serial.println("C");
  Serial.print("  Umid Media : "); Serial.print(somaUmid / 10.0, 1); Serial.println("%");
  Serial.print("  Luz Media  : "); Serial.println(somaLuz / 10);
  Serial.println("───────────────────────────────────────────────\n");
}
